<h2>What is the difference between var, let, and const?</h2>
<p>Answer:</p>
<h3>Var: Old way of declaring variables.</h3>
<ul>
    <li>Not block scoped</li>
    <li>Can be re-eclared</li>
    <li>Can be updated</li>
    <li>Hoisted with undefined</li>
</ul>
<h3>Let: Introduced in ES6 (2015).</h3>
<ul>
    <li>Block scoped</li>
    <li>Cannot be re-declared</li>
    <li>Can be updated</li>
    <li>Not hoisted</li>
</ul>
<h3>Const: Used for variables that should not change.</h3>
<ul>
    <li>Block scoped</li>
    <li>Cannot be re-declared</li>
    <li>Cannot be updated</li>
    <li>Must be initialized when declared</li>
</ul>
<h2>What is the spread operator (...)?</h2>
<p>Answer:</p>
<h3>
    The spread operator (...) in JavaScript is used to expand elements of an array or object into individual elements. It helps to copy, merge, or pass values easily. For example, [...arr] creates a
    copy of an array. It can also combine arrays like [...a, ...b]. This makes code shorter and cleaner.
</h3>

<h2>What is the difference between map(), filter(), and forEach()?</h2>
<p>Answer: </p>
<h3>map(), filter(), and forEach() are JavaScript array methods used to loop through arrays. map() creates a new array by transforming each element. filter() creates a new array containing only elements that match a condition. forEach() simply loops through the array and performs an action, but does not return a new array.</h3>
<h2>What is an arrow function?</h2>
<p>Answer: </p>
<h3>An arrow function is a shorter syntax for writing functions in JavaScript using =>. It makes the code more concise and readable. Arrow functions do not have their own this, they inherit it from the surrounding scope.</h3>
<h2>What are template literals?</h2>
<p>Answer:</p>
<h3>Template literals are a way to create strings in JavaScript using backticks ( ` ` ) instead of quotes. They allow string interpolation, meaning you can insert variables using ${ }. They also support multi-line strings easily.</h3>
